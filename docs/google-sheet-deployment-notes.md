# SCA 課後練習時段部署決策紀錄

## 目前採用

第一階段先採用：

- Google Sheet 當後台資料表
- GitHub Pages 放學生端網頁

原因：

- 先避免付費服務。
- 老師可直接用表格更新每週時段。
- 學生端只需要查看公告，不需要登入或自行預約。
- GitHub Pages 適合放靜態網頁。

## 保留但暫緩

保留 `app/admin.html` 與相關自製後台程式碼，不刪除。

第二階段可升級為：

- 自製後台
- 雲端資料庫
- 單一老師登入
- 更完整的新增、修改、複製、停用流程

暫緩原因：

- 目前不想使用付費主機、付費資料庫或帳號升級。
- Google Sheet 已足夠支援第一階段公告需求。

## Google Sheet 欄位

請建立一個試算表，第一列固定使用以下欄位名稱：

```csv
日期,開始時間,結束時間,義式組狀態,義式組名額,感官手沖組狀態,感官手沖組名額
```

狀態請使用以下三種文字：

- `顯示名額`
- `不顯示名額`
- `不開放`

範例：

```csv
日期,開始時間,結束時間,義式組狀態,義式組名額,感官手沖組狀態,感官手沖組名額
2026-06-22,10:00,12:00,顯示名額,3,顯示名額,2
2026-06-22,14:00,16:00,不開放,0,顯示名額,4
2026-06-23,13:30,15:30,顯示名額,2,不開放,0
```

## GitHub Pages 注意事項

- GitHub 上只放學生端網站程式。
- 不要把密碼、API Key、私人資料放進 GitHub。
- Google Sheet 若要被網站讀取，需發布成可公開讀取的 CSV 或使用 Apps Script 產生只讀資料端點。
